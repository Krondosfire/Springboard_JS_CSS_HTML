let

let is a block scope variable. It is not hoisted. It is not available outside the block.

    Properties of let
  • It can be reassigned but not redeclared (unlike var)
  • The let keyword creates a block-scoped variable: a variable that only exists inside a code block
  • Code Block - anything inside the curly braces:
    "{
    // this is a code block
    let x = 5;
    var y = 10;
    }
    console.log(x);
    // ReferenceError: x is not defined
    console.log(y);
    // 10"

const

    Properties of const
  • it prevents a variable from ever being reassigned or redeclared
  • const is also block-scoped
  • const value can be mutated

Hoisting

    map() method
  • Creates a new array
  • Iterates through an array
  • Runs a callback function for each value in the array
  • Adds the result of that callback function to the new array
  • Returns the new array
  • map() method always returns a new array of the same length

    filter() method:
  • Creates a new array
  • Loops through an array
  • Runs a callback function on each value in the array
  • If the callback function returns true, that value is pushed to the new array
  • If the callback function returns false, that value will not be included in the new array
  • The result of the callback will always be evaluated into a boolean
